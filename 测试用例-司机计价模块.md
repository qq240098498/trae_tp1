# 司机计价模块测试用例

## 一、测试概述

### 1.1 测试目标
对司机计价模块进行全面测试，覆盖夜间加费、附加费（楼层费、搬运费、其他附加费）、里程费、等候费等计价规则，并验证司机里程统计功能的正确性。

### 1.2 测试范围
- 计价模块核心计算逻辑（FreightConfigService）
- 订单创建与完成流程中的计价（OrderService）
- 司机里程统计功能（OrderService.getDriverMileageStats）

### 1.3 测试环境
- 单元测试框架：JUnit 5
- Mock框架：Mockito
- 编程语言：Java 8
- 项目框架：Spring Boot 2.7.18

## 九、关键测试点总结

### 9.1 夜间加费关键点
1. ✅ 时段判断：22:00-06:00包含边界点
2. ✅ 计算基数：起步价+里程费作为加价基数
3. ✅ 最低收费：夜间费有最低保障金额
4. ✅ 跨天处理：凌晨00:00-06:00也属于夜间时段

### 9.2 附加费关键点
1. ✅ 楼层费：按层计算，0层或null不收费
2. ✅ 搬运费：用户输入金额，直接累加
3. ✅ 其他附加费：灵活的额外收费项
4. ✅ 费用明细：每种费用都有对应的明细记录

### 9.3 里程统计关键点
1. ✅ 多维度排序：支持总里程、订单数、收入、平均里程等排序
2. ✅ 数据聚合：正确统计总里程、总收入、总运费
3. ✅ 空值过滤：无订单的司机不出现在统计中
4. ✅ 精度处理：金额保留2位小数，里程保留2位小数
